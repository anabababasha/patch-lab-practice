# AGENTS.md — PatchLab Agent Constitution

**Read this entire file at the start of every session, before touching anything.** It exists because agents have repeatedly invented APIs, deleted working code, claimed verification that never happened, and broken enumerations. Every rule below was paid for with a lost session.

This file mirrors the architect-side project skill. Agents do not edit this file unless the build prompt explicitly says so.

---

## 1. ABSOLUTE PROHIBITIONS

1. **NEVER commit or push.** The maintainer verifies in a real browser by ear and eye, then commits personally. No exceptions, no "helpful" commits, no `--amend`, no force-push.
2. **NEVER `git add .`** — staging is always explicit, file by file, and it is the maintainer's job anyway.
3. **NEVER delete or wholesale-rewrite a file** unless the build prompt names that file for deletion/rewrite. No "cleanup," no removing "unused" code, no reformatting passes, no license-header sweeps. Targeted edits only.
4. **NEVER touch** `vite.config.ts` `base`, or `.github/workflows/deploy.yml`.
5. **NEVER edit `src/patterns/`** (and future `src/tunings/`) unless the prompt explicitly targets them. This is cited cultural data — curated, versioned, sourced. Code agents do not author culture.
6. **NEVER invent names.** Before calling any helper, store action, service method, CSS token, or component: grep for its definition. If it does not exist, STOP and say so. (Past inventions: `setUi`, `startAudio`, `DesignNode`, `fader`.)
7. **NEVER claim verification you did not perform.** "Build passes" ≠ "works." Say exactly what was and was not tested, and how.
8. **NEVER exceed the build prompt's scope.** No drive-by refactors, no bonus features. If you see a problem outside scope, report it in the session report.
9. **NEVER include the maintainer's real name** in code, comments, commits, or docs.
10. Work scratch/profiles in `D:\tmp`, never inside the repo.

## 2. ROLES AND GATES

- **Architect (Claude):** writes build prompts and owns architecture decisions. Build prompts arrive as markdown files; they are the sole source of scope.
- **Agent (you):** implements exactly the prompt. You may run `npm run build`, a dev server, and headless-browser repros.
- **Maintainer:** runs all git, performs browser sign-off, owns the ears. **No code ships without maintainer browser sign-off.**
- Every session ends with a **SESSION REPORT** (format in §6). No report, no done.

## 3. SESSION PROTOCOL

**Preflight (mandatory, report verbatim):**
1. `git status --short` and `git log --oneline -1`. Bug-fix rounds may run on the declared dirty state; new feature builds require a clean tree — if dirty when cleanliness is required, STOP and tell the maintainer.
2. State which port serves the current working tree (this machine grows orphaned Vite servers; stale-server repros have burned sign-offs). Detached Vite on Windows: launch with `CI=true`.
3. Cite the TopBar **build stamp** (`v{version} · dev {HH:MM:SS}`) in every repro claim — the dev time is the SERVER START time; an old stamp means an old server.
4. Preflight output must be a FRESH run. Repeating a previous session's output verbatim = not executed.
5. If your terminal cannot run `npm run build`, say so in the FIRST LINE of your report and switch to manual mode (you write commands/probes, the maintainer runs them). NOTE: terminal sandboxing is currently broken on Windows — manual mode is the NORM; write your verification section as exact runnable commands, and expect the maintainer to run `npm run build`, `npm run test:corpus`, and any worklet specs before reading the rest of your report.

**While working:**
- Read the ENTIRE file before editing it. Half-read files caused half-refactors and duplicate method bodies.
- For bugs: **instrument before fixing.** Add temporary probes, observe the actual mechanism, then fix, then remove probes. Plausible-but-wrong root causes have shipped before; evidence or it didn't happen.
- `npm run build` must pass before you report done.

## 4. ARCHITECTURE CONTRACTS (load-bearing — violating these breaks the instrument)

- **Store-as-truth + sanitize gate.** All design state flows through the typed store; `sanitizeDesign` strips anything not declared in the registry — including hidden params. Every param, visible or hidden, must be registered or it silently dies on import.
- **AudioUnit contract.** Structural changes go through the debounced (50ms) **incremental splice rebuild**: kept units stay alive and connected; only created units are built and only removed units are disposed (their optional `prepareTeardown(when)` pre-fades fire first so teardown is silent); wires diff by key; full rebuild survives only as the guarded fallback. `console.debug('[rebuild] splice', …)` logs every pass — cite its counts. Param changes go through `bind()` only — never rebuild for a param.
- **Transport clock is immortal.** It never resets on rebuilds; sequencer/looper phase DERIVES from the global tick — never unit-local counters. Update a playback anchor BEFORE changing speed or restarting a source.
- **Service-owned sources never stop during rebuilds** (`entry.source`/`entry.bus` in looperService) — they play into a momentarily disconnected bus and the new unit reconnects; `reattachPlayback` is a DELIBERATE no-op. A unit must be trigger-ready from creation. The Looper capture input and Master mute gains init at 0 and ramp in; do not "fix" that.
- **Worklets preload eagerly** in `engine.start()` — never `await addModule` inside a rebuild path.
- **DSP worklet law** (`grainWorklet.ts` and `bufferRepeaterWorklet.ts` are the templates): inline template-literal source via Blob URL; ZERO allocation in `process()` (flat typed-array pools, LUTs built in the constructor, no closures); one-pole in-worklet smoothing; params via `port.postMessage`, engine pre-resolves tempo sync. Freeze-type features stop the write HEAD, not just writes (an advancing head slides read-windows into stale ring). Wet buses are BOUNDED: overlap compensation then tanh; feedback taps the bounded signal. Every DSP worklet ships with OfflineAudioContext render tests in `tests/worklets.spec.ts` (non-silence, state-holds-vs-control, bypass transparency, feedback stability, and — since 8-3q — click/artifact detectors calibrated to fail on the pre-fix worklet) — they run in the corpus harness via the `window.__plWorkletTest` hook in main.tsx and are ACCEPTANCE CRITERIA, not optional: they caught real DSP bugs that compile+review missed.
- **DSP worklet law, part 2 (8-3q hardening — apply to every future voice/burst/grain worklet):**
  - A pitched-up read must never cross the write head: glide-behind while writing; WRAP into the spawn window while frozen (never pin-and-hold).
  - Smooth the pre-computed target scalar — never recompute an expensive function from a raw value every sample.
  - Drop-over-steal: a full voice pool silently skips the new voice rather than cutting a live one.
  - Deferred-apply for retriggers: stash the incoming message, cosine-fade the live output (~256 samples), apply only once faded or the target time arrives — never stomp live state in one sample.
  - Ring-buffer lap auto-release: any hold/gate/freeze bounded by ring capacity must auto-release at a faded boundary before the write head can lap it.
  - Absolute-frame ring indexing: `(absF-baseFrame)&mask`, `baseFrame` captured once on first `process()` — never a private incrementing head.
- **Tempo Sync resolves ENGINE-SIDE only** (`src/audio/sync.ts`, one helper: `resolveParamValue`). Units never see divisions — they receive plain Hz/ms. Syncable params carry a REGISTERED hidden `{id}_div` param (0=Auto, 1=Free, 2+=division); explicit Free/division always wins, Auto follows `design.settings.sync`. The stored free value is never overwritten. Do not add sync logic inside units.
- **`Design.settings` is optional-additive** (`{ bpm?, sync? }`): sanitize validates and "absent stays absent" — pre-5d designs must round-trip byte-identically FOREVER. `setBpm` persists without an undo snapshot (accepted edge: design-bpm can lag live bpm after undo).
- **`engine.trackDesign` invariant**: engine `lastDesign` is structural-fresh only; any engine feature reading design state on the PARAM path requires the store to call `engine.trackDesign(design)` before `setParam` (writeParam/setParamsBulk already do). New param-path design readers must keep this.
- **Teardown is guarded per node.** Every `dispose()` uses bare `.disconnect()` in try/catch per node; the engine teardown loop try/catches per unit so one bad dispose can never orphan the rest. Do not remove these guards.
- **Onset-declick law:** every voice/grain START ramps gain 0→1 over ~3ms (`setValueAtTime(0,time)` + `linearRampToValueAtTime(1,time+0.003)`), scheduled at the voice's own start time — mirrors the existing stop-side 3ms declick. Sampler's `hitGain` is the template for any future voice-triggered component.
- **Sampler voice bookkeeping under lookahead:** a scheduled-but-not-yet-started voice must stay tracked in the active set until it truly starts or is cancelled — removing it once a stop is merely scheduled breaks a later choke/mute's ability to cancel it, producing the "choke stops working until refresh" bug class.
- **Three wire kinds** (audio / control / trigger); fan-in forbidden at inputs; dedup keys on destination pin only; `isValidConnection={() => true}` stays on `<ReactFlow>`.
- **Trigger bus** carries time-stamped events on a lookahead transport. (Roadmap: events gain `phase: 'on'|'off'` — default `'on'`, fully backward compatible.)
- **Services** (media/mic/recorder/looper/eq/meter/scope/midi) own anything that must survive rebuilds; engine calls `prune(liveIds)` post-rebuild; service→store communication via callbacks assigned at the store bottom. **Static imports only** between store/services/units.
- **Looper contract:** Output carries loop playback ONLY. Empty/stopped = silent. Dry monitoring is the patcher's job via fan-out. Capture taps the input regardless.
- **Looper Play Sync contract:** per-looper `playSync` toggle, default off. On, PLAY and speed-changes quantize to bar boundaries via a `pendingAction` consumed synchronously inside the shared transport tick callback (never `setTimeout`). Phase derives from `tickIndex` via an absolute anchor (bar 0 = transport start) — any two synced loops agree by construction.
- **Record-arm sample-accurate scheduling:** Bar-Sync record-start calls `tap.armAt(time)` — `pl-capture` now uses an absolute-frame `arm_at`/`atFrame` gate (pioneered by `pl-bufferrepeat`) instead of `setTimeout` + immediate `arm()`. Two independent Loopers fed the identical time from one shared tick callback land on the exact same sample, zero coordination needed. Rule: only convert a pending-action to synchronous/absolute-frame scheduling when two+ nodes must agree on the SAME instant; `stop_record` stays on `setTimeout`.
- **Numeric editing law:** free-text numeric inputs stage locally; commit on Enter/blur; Enter blurs; Escape reverts; empty/non-finite reverts; clamp to registry min/max; one edit = one undo step. The engine-level finite guard (reject non-finite in the bind path) is permanent.
- **Any cross-component communication channel must be nodeId-scoped.** (The on-node keyboard's DOM event `pl-midi-pointer-note` carries nodeId; any future channel must too.)
- **Activity-aware visualization** is the fourth data-flow pattern: node/wire liveness reads from `meterService`'s existing rAF sampling, exposed via a module-level Map + `useSyncExternalStore` (never Zustand, which would re-render the tree at 60fps). Trigger-only nodes mark activity explicitly at `engine.emitTrigger()`.
- **Enumerations are complete or the feature is broken.** A new component/param/display type goes into EVERY one of: `paletteOrder`, the `display` union AND its PinTableNode branch, validation rules, help content, examples index, registry (sanitize derives from it). Forgotten three times; grep them all.
- React/UI: ALL hooks before any early return (three crashes); `min-width: 0` on flex children containing range inputs; per-note/per-step visual flashes via classList on refs, never React state; interactive elements on nodes carry `nodrag`; CSS via tokens.css tokens only; follow docs/DESIGN.md (desaturated at rest).
- **React Flow default styling + hard-bound gestures:** audit the library's own default CSS for a state class before assuming custom styling has a bug (`.react-flow__node.selected`'s default box-shadow stacked unnoticed). A declarative gesture prop (e.g. `selectionOnDrag`) can be hard-bound to a specific mouse button at the implementation level — verify against the installed package's actual source before promising a config-only rebind.
- Example-builder param ids must match the registry EXACTLY (sanitize silently drops mismatches — watch its console.warns).

## 5. HARD-WON LESSONS (terse; full history lives architect-side)

1. One throwing dispose used to abort all teardown → stacking "polyphony." Loop-level guard is the durable fix.
2. HMR/stale-server desync produces phantom bugs — verify the served port before trusting any repro.
3. Invented identifiers: grep first, always.
4. Conditional hooks crash React — order violations found the hard way, three times.
5. Enumeration omissions make components exist-but-unplaceable.
6. Hidden params die at the sanitize gate unless registered.
7. "Verified" claims without a browser are worthless; say what you actually ran.
8. TypeScript's DOM lib already ships Web MIDI types — use native `MIDIAccess`/`MIDIInput`/`MIDIMessageEvent`.
9. React effect destructors must return void — wrap `Set.delete()`-style unsubscribers.
10. Detached Vite on Windows dies on stdin close — `CI=true`.
11. Selects stored as numeric indices resolve external ids in their owning service; document the mapping and handle device-list reordering.
12. Web Audio: a started `AudioBufferSourceNode` has acquired its buffer — edits are inaudible until playback restarts with the swapped buffer. Every buffer-shaping op must restart through the CURRENT output path.
13. GH Pages deploys occasionally fail platform-side — remedy is an empty-commit retry (maintainer runs it).
14. A fix shipped on simulation alone became the next session's bug (looper reattach) — runtime/ear evidence gates every mechanism claim.
15. Momentary controls lose to event streams: a momentary voice-stop is inaudible under a running sequencer (~150ms re-trigger). Controls that defeat trigger streams must be stateful (mute/toggle). "X doesn't work at all" often means "X worked and was instantly re-triggered" — diagnose the stream, not the button.
16. Registry help strings hold real Unicode — patch from file bytes, never terminal echo (mojibake incident).
17. Headless Web Audio probes: Chrome needs `--autoplay-policy=no-user-gesture-required`, and wrap `engine.start` in an explicit timeout so a blocked start identifies itself.
18. Known accepted edges (do not "fix" silently): live-wire deletion may click; the Recorder tap still records rebuild gaps (queued fix, same pattern as the Looper input); two separate delete gestures inside one 50ms debounce — the second misses its pre-fade; design-bpm may lag live bpm after undo (BPM writes skip history by design).
19. Dependency-free CDP probes work (Node ≥ 22 global WebSocket): dev-mode `import('/patch-lab-practice/src/…')` reaches the app's LIVE module singletons — drive the real store/engine and measure signals (an LFO period measured against BPM is gold-standard evidence). Keep harnesses in `D:\tmp`.
20. Spec-formula compliance isn't proof of correctness if the spec's own model was incomplete — ask whether the acceptance test would even catch the failure mode the spec missed.
21. Precision state that accumulates unboundedly (absolute-frame counters) needs `Float64Array` — `Float32Array` silently loses whole-frame precision after ~12 minutes of uptime, invisible in short tests.
22. Deferred/pending-state machines need every interleaving enumerated at design time: a second event while one's pending, a cancel for something not yet applied. A live performer finds every one within minutes.
23. Controlled synthetic-signal probes beat real-file repros for DSP/click mechanisms: a test signal with a mathematically known exact expected value, compared to many significant figures, proves the mechanism and rules out competing explanations a real file can't.
24. A live probe harness can have its own bugs indistinguishable from real findings (a duplicated async setup racing a second instance into existence; forgetting to start something the feature depends on). When a probe result surprises you, audit the probe's own setup first.
